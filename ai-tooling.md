# AI Tooling

## AI Tools Used

The primary AI tool used during development was ChatGPT.

ChatGPT was used as an AI pair-programming assistant throughout the project lifecycle.

## How AI Was Used

### Project Planning

AI assistance was used to:

* Interpret project requirements
* Break the assignment into manageable steps
* Select an overall RAG architecture
* Identify milestones and implementation order

### Code Generation

AI assistance was used to generate and refine:

* Document ingestion code
* Chunking logic
* Retrieval functions
* Prompt construction
* Groq API integration
* Flask web application code
* Automated tests
* Evaluation scripts
* GitHub Actions workflow

### Debugging

AI assistance was used to diagnose and resolve:

* Python environment issues
* Package installation problems
* File path errors
* API authentication issues
* Flask application errors
* Testing and CI configuration issues

### Documentation

AI assistance was used to help draft:

* README documentation
* Design and evaluation documentation
* AI tooling documentation

## What Worked Well

* Rapid prototyping of the RAG pipeline
* Debugging Python and environment issues
* Generating boilerplate Flask code
* Creating evaluation datasets
* Explaining RAG concepts and implementation decisions

## Challenges

* Dependency installation issues on macOS
* Package compatibility problems with embedding and vector database libraries
* API authentication troubleshooting

## Reflection

Using AI significantly accelerated development by reducing time spent on boilerplate code, troubleshooting, and documentation. AI was most effective when used as an interactive development partner rather than as a fully autonomous coding solution. All generated code was reviewed, tested, and modified as necessary before inclusion in the final project.
